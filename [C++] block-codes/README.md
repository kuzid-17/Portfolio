# Моделирование работы блочных кодов в PUCCH FORMAT2

Программная реализация логики работы блочных кодов на языке высокого уровня C++

## Зависимости

Перед сборкой необходимо установить библиотеку nlohmann/json.
```bash
sudo apt install nlohmann-json3-dev
```

## Сборка

Проект собирается при помощи Makefile. Для сборки используется следующая команда:
```bash
make
```

Исполняемый файл `pucch_f2` появится в корневом каталоге.

Для очистки:
```bash
make clean
```

## Запуск

Программа принимает на вход путь до JSON-файла с параметрами запуска:
```bash
./pucch_f2 <path_to_file>.json
```

Результат записывается в `result.json`.

## Формат входных данных

Поддерживается три режима работы: кодирование, декодирование и симуляция канала с АБГШ.
Все сценарии принимают JSON-файл с обязательным полем `"mode"`.

### Кодирование
```json
{
    "mode": "coding",
    "num_of_pucch_f2_bits": /* 2, 4, 6, 8 или 11 */,
    "pucch_f2_bits": /* массив бит */
}
```

Результат — массив QPSK-символов.

### Декодирование
```json
{
    "mode": "decoding",
    "num_of_pucch_f2_bits": /* 2, 4, 6, 8 или 11 */,
    "qpsk_symbols": /* массив комплексных чисел формата "0.536+0.357j" */
}
```

Результат — массив декодированных бит.

### Симуляция канала
```json
{
    "mode": "channel simulation",
    "num_of_pucch_f2_bits": /* 2, 4, 6, 8 или 11 */,
    "iterations": /* количество итераций */,
    "noise_level": /* уровень шума */
}
```

Результат — значение BLER, количество успешных и ошибочных передач.

Все поля соответствуют требованиям задания.
`"noise_level"` — дополнительное поле, задающее параметр канала: чем больше значение, тем лучше качество передачи.

## Результаты моделирования

В рамках проекта была проведена симуляция работы блочных кодов для PUCCH FORMAT2 в канале с АБГШ.  

Для каждого набора параметров (длина вектора данных, уровень шума, количество передач) вычислялся **BLER** (block error rate) и на основе полученных данных были построены графики, демонстрирующие зависимость вероятности ошибки на блок от параметра канала для разных кодов. Графики сохранены в папке `plot` и могут быть использованы для визуального сравнения кодов между собой. Каждый график подписан соответствующими параметрами моделирования.
